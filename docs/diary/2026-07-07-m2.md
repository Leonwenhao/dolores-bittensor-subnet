Completed M2 validator core per the Fable testnet development plan: subnet

## Changes

- Added ordered pre-gates in `src/dolores_subnet/gates.py`:
  `schema_version`, `size`, `parse`, `hash_match`, `quota`, `epoch_duplicate`.
- Added `src/dolores_subnet/bridge.py`:
  - Calls `dolores.pipeline.run_task_pipeline(...)` with
    `backend="docker"`, `mode="generated"`, `allow_docker_fallback=False` in
    offline validator mode.
  - Classifies infra errors exactly from `executed=False`,
    `fallback_reason != None`, and no safety findings.
  - Purges archive rows for infra errors.
  - Separates safety rejections from failed-reference rejections.
  - Emits runtime-cost-free `task_value`.
- Added `src/dolores_subnet/archive.py` for archive initialization,
  `submissions.jsonl`, `purge_task`, and public-safe archive copies with hidden
  tests removed.
- Added `scripts/seed_adversarial.py`, generating committed fixtures under
  `tests/fixtures/planted/`.
- Rewrote `neurons/validator.py` with the M2 offline submissions-directory path.
- Added tests for gates, bridge classification/purge behavior, local fixture
  validation, and hidden-test removal from public archive copies.
- Extended scoring helpers with `task_value_from_score` and deterministic
  normalization.

## Deviations / Notes

- The installed Dolores `DockerRunner` tried to discover
  `docker/verifier/Dockerfile` relative to the venv. The bridge now sets
  `DOLORES_VERIFIER_DOCKERFILE` to the Dolores repo Dockerfile before Docker
  validation. No Dolores backend code was modified.
- `docker image inspect dolores-verifier-pytest:0.1.0` had previously been
  inconsistent on this daemon, so the verifier image was rebuilt from the
  Dolores Dockerfile with:
  `cd "$DOLORES" && docker build -f docker/verifier/Dockerfile -t dolores-verifier-pytest:0.1.0 .`
- `scripts/seed_adversarial.py` selects deterministic task IDs by stable-hash
  ordering so `good_parser` validates before duplicate-like bad packages. This
  prevents safety/failed bad fixtures from poisoning the good package's archive
  duplicate report during the M2 planted-set run.
- `verification.error_summary` is normalized at the bridge boundary to replace
  pytest elapsed-time text with `in <duration>s`, keeping `submissions.jsonl`
  deterministic while preserving the failure class.
