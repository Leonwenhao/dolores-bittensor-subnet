## Deviations / Notes

- This Python build skips hidden `.pth` files, and setuptools created an
  editable-install path file named `__editable__...pth`. That made
  `scripts/preflight.py` unable to import `dolores_subnet` when executed by path.
  The script now prepends this repo's `src/` directory before importing the
  local package. This is limited to the subnet script bootstrap and does not add
  Dolores backend source paths.
- Docker listed `dolores-verifier-pytest:0.1.0` and `docker run` resolved it, but
  `docker image inspect dolores-verifier-pytest:0.1.0` did not resolve the tag on
  this daemon. `preflight.py` first tries inspect, then falls back to exact
  `docker image ls` tag detection. The container smoke confirms the image is
  runnable.
- The Dolores source directory does not expose a usable Git timestamp on this
  machine, so the install-freshness check reports the installed package is
  present and the source timestamp is unavailable.
