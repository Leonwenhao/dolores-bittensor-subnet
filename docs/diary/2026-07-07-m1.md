Date: 2026-07-07
Branch: `m1-packaging`

## Scope

Completed M1 packaging and wire-format implementation per the Fable testnet
development plan.

No wallets were created or read, no chain actions were performed, no GitHub push
was attempted, and no paid provider calls were made.

## Changes

- Added `protocol.SCHEMA_VERSION` and `protocol.WireSubmission`.
- Added `src/dolores_subnet/packaging.py`:
  - `to_wire(task)` emits the v0 wire dict with Dolores canonical package JSON
    and `TaskPackage.stable_hash()`.
  - `from_wire(payload)` validates schema version, size, parseability, task id,
    and claimed hash.
  - `materialize(task, root)` delegates to Dolores `write_task_package`.
  - `wire_size_ok(payload)` and `WireError(reason, detail)` provide gate-ready
    machine-readable failures.
- Added tests for:
  - Generated `parser_roundtrip` package round-trip hash stability.
  - Disk materialization parity with `TaskPackage.load`.
  - Tamper/hash rejection.
  - Oversize rejection at the 200 KB M0 cap.
  - Distinct malformed-payload reason codes.
  - Cross-process deterministic hash output.

## Deviations / Notes

- The separate-process determinism script uses `PYTHONPATH=src` because this
  macOS/Python environment repeatedly marks pip-generated `.pth` files in the
  venv as hidden, and the interpreter skips hidden `.pth` files. The package
  logic itself is not affected; the deterministic hash values matched exactly.
- Hashing is delegated to Dolores `TaskPackage.stable_hash()`; no subnet-side
  hash reimplementation was added.
- Disk layout is delegated to Dolores `write_task_package()`.
